# Version 9.27.1

# Major New Features

- The following features are now available for all users:
  - Ability to set a 'Pick Date' and 'Pick Time' for [fruit bookings](https://support.vintrace.com/hc/en-us/articles/32303268370324-Managing-Fruit-Intakes-and-Fruit-Intake-Bookings#h_52356d83-df02-4dfe-af54-839f2af65849)
  - An additional tile on the wine detail page showing the allocated Product/s
  - The ability to filter the [Crush/Press Loads console](https://support.vintrace.com/hc/en-us/articles/32303300235796-Crush-Press-Loads) by Winery
  - An additional 'Multi-step' option for the Fruit Delivery Mode system defaults
  - The ability to filter the [Fruit Intake console](https://support.vintrace.com/hc/en-us/articles/32303330881044-Fruit-Intake-Console) by Carrier
  - A [Spirits Stock report](https://support.vintrace.com/hc/en-us/articles/32301303165588-Spirits-Stock-Report) for liquids in a Spirits Tax Class
  - Warnings when scheduling operations that will blend bonded and tax paid wines
  - Distilled Spirits Plant[(DSP) functionality](https://support.vintrace.com/hc/en-us/sections/32300840834452-Distilled-Spirits-Plant)
  - Ability to [change a juice to a concentrate](https://support.vintrace.com/hc/en-us/articles/32301281778708-Evaporating-Juice-into-Concentrate-and-Reconstituting-Concentrate)
  - Ability to create and report on [Formula Wine](https://support.vintrace.com/hc/en-us/articles/32301298832276-Flagging-a-Wine-as-a-Formula-Wine)
  - The 'Harvester' field will populate on fruit bookings from the Grower/Block field
  - Additional options for 'Product state' and 'Program' when [creating Auto-Code Policies](https://support.vintrace.com/hc/en-us/articles/32303292885908-Configuring-and-Using-Auto-Codes) for Type 'Batch'

# Additional Fixes and Improvements

- Fixed an issue causing a blank grey dashboard when language is set to Italian
- Added 'In production' and 'Supply' as reference columns to the product allocation import/export
- Fixed an issue where 'Days on skins' not showing for Bins
- Fixed an issue where some work orders were not displaying on the Work Order Detail report
- Fixed an issue causing some incorrect values in the 'To Vessel Start Batch' column on the Operation History report for 'EXCEL - Single line detail report'
