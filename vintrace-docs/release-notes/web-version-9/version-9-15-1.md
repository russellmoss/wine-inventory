# Version 9.15.1

# Additional Fixes and Improvements

Not all of the new features are enabled by default. If you would like to use any of these features, please contact our support team.

- Added the vessel forecasting functionality to the Analysis and Treatment (Product) operations
- Added the ability to add jobs to work orders that are in progress
- Updated the summary in the history tab when importing products/allocations
- Fixed the incorrect messaging on the tooltip displayed on the vessel forecasting status
- Fixed an issue affecting vessel contents when editing jobs with vessel forecasting
- Added the ability to add a product to a block assessment without the "Can Add/Edit Allocation Products" permission
- Added the Bill of Lading number to the v7/operations/shipments API
- We updated a column header when viewing grower contracts from 'Tonnage' to 'Area' depending on the selected cost metric
- Fixed a bug where the intake's fruit cost type was not showing on a grower contract
- Fixed a bug where the TTB report was incorrectly returning as empty
- Fixed a bug causing stock item discrepancies
- We updated the headers on the Inventory Stock Report
- Fixed a bug causing incorrect units to transfer from Sales Orders to Invoices
- Fixed a bug causing the wrong default for the historical vessel exposure year field
- Fixed a bug that affected closing work order jobs on bins which contain wine
- Updated the Bulk Cost Summary report to include all cost buckets
