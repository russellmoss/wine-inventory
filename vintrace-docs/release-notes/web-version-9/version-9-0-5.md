# Version 9.0.5

# vintrace

## Major new features

- [Ability to manage stock allocations for cased goods](https://support.vintrace.com/hc/en-us/articles/32301321793812)
- [Ability to generate a fruit booking report with scheduled extraction vessels](https://support.vintrace.com/hc/en-us/articles/32301353043348)

## Additional fixes and improvements

- Added the Stack location to the Vessels page in the new vintrace
- Fixed an issue to extend the For Use date on 702 reports to 2020
- Fixed an issue that caused sales orders' discount amounts to be incorrectly calculated
- Fixed an issue that caused the Product to display the Vintage and Product instead of the Product name
- Fixed an issue that caused the Harvest Method to be updated incorrectly for a booking if there is a matching assessment with a Harvest Method
- Fixed an issue that prevented users from submitting the fruit receival on the app if the total amount is higher than the scale's capacity for bin and truck scale mode
- Fixed an issue that caused the activity summary report to be out of balance for press cycles with a gain or loss
- Fixed an issue that prevented users from viewing scheduled extraction work orders in the app if fruit has not been delivered yet
- Fixed an issue where the unit of Specific Gravity was reported as Baume, not SG
- Fixed an issue that caused the received value in Part IV of the TTB report to be different from the Operation Throughout Report
- Fixed the issue that caused the winery defaults to be cleared out
- Fixed an issue that caused the My Searches dashlet to display an error
- Fixed an issue that prevented users from requesting lab work for bins on the Vessels page
