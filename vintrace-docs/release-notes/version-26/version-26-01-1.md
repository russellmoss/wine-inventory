# Version 26.01.1

Version roll-out dates: Mon, 5 Jan - Mon, 12 Jan 2026

## General availability

*The updates in this section are generally available to all customers as long as the module in question is within the customer package / licensing scope.*

### Wine operations

- Additional performance improvements were made to the following transfer operations to continue to bring you a better user experience.

  - Extraction
  - Multi topping
  - Multi transfer (many-to-one)
  - Multi transfer (one-to-many)
  - Press cycle
  - Transfer/Rack/Blend

### Vessels module

- An issue has been fixed that prevented the tasting sheet from being generated from the Vessels page > Actions > Tasting sheet.
- The Yield label was updated as it was incorrectly displaying the rate unit for US and metric unit customers.![Wine Product Page Yield Tile 20260109.jpeg](https://support.vintrace.com/hc/article_attachments/45193312323732)

### Reporting and analytics module

- An issue was fixed for the [Air Displacement Report](https://support.vintrace.com/hc/en-us/articles/32301303790996-Air-Displacement-Report "https://support.vintrace.com/hc/en-us/articles/32301303790996-Air-Displacement-Report") that resulted in the transactions appearing on the incorrect date.

### Vineyard management module

#### Work tracking

**GPS tracks Analysis**

Added “Filter work” option.

To the Analysis feature of the Work tracking module, we have added an option to filter work/GPS tracks by time, vineyard block, work type, users, machines, and/or vine variety, similar to the Filter work option in the Work tracking feature.

#### Calendar

**Added new unit of measure to Harvest and Ice harvest tasks**

To the Harvest and Ice harvest tasks, a unit of measure, “lb,” for US customary units was added. With that, the Documentation reports and the Harvest quantity on the vineyard block (in the Cost analysis module) were also updated to reflect the newly added unit.

#### Vineyard block properties

**Added customizable “Block type” property**

A Block type property was added to every block, along with the capability for every organization to add custom block types. As a result, activities and properties related to non-vineyard blocks can now be tracked in the system, such as activities related to biodiversity areas, garage or service areas, as well as vineyard blocks can be categorized into various classes such as productive, newly planted, etc. Furthermore, some analysis and/or reporting features, such as for example GPS tracking analysis, can already make use of Block type and allow segmenting of work records per Block type. More features will make use of Block type in the future.

#### General updates

**vintrace logo, font, and name unification.**

We have updated the font styles to match global vintrace font, creating a more consistent and unified visual experience across the solutions. For users coming through vintrace login, the solution name has been changed to "vintrace Vineyard management", and the vintrace logo is shown rather than eVineyard.

**Improved localization capability**

Reports, available vine varieties, available default vineyard activities and several other specifics across the application, related to the country of vineyards, have been made dependent on the organization’s country, and decoupled from the selected user’s language. Result is improved localization of the application across various countries using English language, and improved cross-language consistency for organizations where users may be using application in one country but in various languages.

#### Other updates

- On top of that, several fixes, improvements, and minor upgrades have been completed in the January release.
